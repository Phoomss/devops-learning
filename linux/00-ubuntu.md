# 🐧 UBUNTU 101 – คำสั่งพื้นฐานที่ควรรู้

## 1. 📁 การจัดการไฟล์และโฟลเดอร์

| คำสั่ง | คำอธิบาย | ตัวอย่าง |
|--------|-----------|----------|
| `ls` | แสดงไฟล์ในโฟลเดอร์ | `ls -la` |
| `cd` | เปลี่ยนไดเรกทอรี | `cd Documents` |
| `pwd` | แสดง path ปัจจุบัน | `pwd` |
| `mkdir` | สร้างโฟลเดอร์ | `mkdir my_folder` |
| `touch` | สร้างไฟล์เปล่า | `touch file.txt` |
| `rm` | ลบไฟล์ | `rm file.txt` |
| `rm -r` | ลบโฟลเดอร์แบบ recursive | `rm -r my_folder` |
| `cp` | คัดลอกไฟล์ | `cp file1.txt file2.txt` |
| `mv` | ย้ายหรือเปลี่ยนชื่อไฟล์ | `mv old.txt new.txt` |

## 2. 🔍 คำสั่งค้นหาและดูเนื้อหา

| `find` | ค้นหาไฟล์ | `find . -name "*.txt"` |

# 📂 โฟลเดอร์หลักใต้ `/` (Root Directory) ของ Linux

ในระบบ Linux โฟลเดอร์ที่อยู่ในระดับบนสุดของโครงสร้างไฟล์จะอยู่ภายใต้ `/` ซึ่งเป็น root ของระบบไฟล์ทั้งหมด

---

## 🔍 คำอธิบายแต่ละโฟลเดอร์

| โฟลเดอร์ | คำอธิบาย |
|----------|-----------|
| `/` | รากของระบบไฟล์ (root directory) |
| `/bin` | ไฟล์คำสั่งพื้นฐาน (binary) ที่จำเป็นต้องใช้ตอนบูต เช่น `ls`, `cp`, `mv` |
| `/boot` | ไฟล์ที่จำเป็นในการบูตระบบ เช่น `vmlinuz`, `initrd`, `grub/` |
| `/dev` | อุปกรณ์ของระบบ (device files) เช่น `/dev/sda`, `/dev/null` |
| `/etc` | ไฟล์ config ของระบบและโปรแกรม เช่น `/etc/hostname`, `/etc/ssh/sshd_config` |
| `/home` | โฟลเดอร์ส่วนตัวของผู้ใช้ทั่วไป เช่น `/home/phoom` |
| `/lib`, `/lib64` | ไลบรารีที่จำเป็นสำหรับคำสั่งใน `/bin` และ `/sbin` |
| `/media` | จุดเมานต์ของ USB/CD/DVD ที่เชื่อมต่อแบบอัตโนมัติ |
| `/mnt` | จุดเมานต์ชั่วคราวที่เราสร้างเอง เช่น mount external disk |
| `/opt` | ติดตั้งแอปพลิเคชันเพิ่มเติมแบบ manual (third-party software) |
| `/proc` | Virtual filesystem ที่ใช้ดูข้อมูล kernel/process เช่น `/proc/cpuinfo` |
| `/root` | โฟลเดอร์ส่วนตัวของ root user (เหมือนกับ /home แต่ของ root) |
| `/run` | ไฟล์สถานะชั่วคราวของ systemd ที่ใช้ในขณะบูต |
| `/sbin` | คำสั่งสำหรับ system admin เช่น `reboot`, `shutdown`, `fsck` |
| `/srv` | ไฟล์สำหรับให้บริการ เช่น FTP, HTTP, Web Server (optional) |
| `/sys` | Virtual filesystem แสดงข้อมูล hardware แบบ dynamic (เกี่ยวกับ kernel) |
| `/tmp` | ไฟล์ชั่วคราว (ลบเมื่อ reboot) เช่น session files |
| `/usr` | ไฟล์โปรแกรมและไลบรารีของผู้ใช้ เช่น `/usr/bin`, `/usr/lib` |
| `/var` | ไฟล์ที่เปลี่ยนแปลงตลอด เช่น log (`/var/log`), spool, cache |

---

## 🧠 ความเข้าใจเสริม

- `/` คือ **จุดเริ่มต้นของทุก path** – ไม่มี drive letter แบบ Windows (เช่น C:, D:)
- Linux จัดโครงสร้างแบบ **Filesystem Hierarchy Standard (FHS)** ที่ทุกอย่างคือไฟล์
- โฟลเดอร์อย่าง `/bin`, `/sbin`, `/lib` มีความสำคัญมากในช่วงบูตระบบ
